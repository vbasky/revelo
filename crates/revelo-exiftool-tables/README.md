ExifTool-derived maker-note tag tables for 14 camera vendors, used by
[revelo](https://github.com/vbasky/revelo) to reach ExifTool-grade depth in
maker-note decoding.

Part of the [**revelo**](https://github.com/vbasky/revelo) project — see the
[project README](https://github.com/vbasky/revelo#readme) for the full
picture.

## What it provides

The crate exposes two kinds of lookup for each supported vendor:

- **Tag name** — maker-note tag id → canonical ExifTool tag name.
- **PrintConv** — (tag id, integer value) → ExifTool human-readable string.

These are the only two things ExifTool's tag tables really are: a name mapping
and an enumerated value mapping. The tables are generated from ExifTool's Perl
source by `codegen/extract.pl` and committed as static Rust arrays.

For Canon and Olympus there are additional vendor-specific helpers for
sub-tables (Canon's `ProcessBinaryData` sub-IFDs and Olympus's type-2
sub-IFDs). For Nikon there is an `AFInfo` sub-table helper. The crate also
provides `exif_name_alias` and `canon_main_name_alias` to reconcile the few
cases where ExifTool's tag names diverge from the standard EXIF names.

## Vendor coverage

| Vendor | `Vendor` variant | Notes |
| --- | --- | --- |
| Apple | `Vendor::Apple` | iPhone / iPad maker notes |
| Canon | `Vendor::Canon` | Main IFD + 12 `ProcessBinaryData` sub-tables |
| Casio | `Vendor::Casio` | |
| DJI | `Vendor::Dji` | |
| FLIR | `Vendor::Flir` | Thermal cameras |
| Fujifilm | `Vendor::Fujifilm` | Also matched on `FUJI` in Make string |
| Konica-Minolta | `Vendor::Minolta` | Matched on `MINOLTA` or `KONICA` |
| Nikon | `Vendor::Nikon` | Main IFD + AFInfo sub-table |
| Olympus | `Vendor::Olympus` | Main IFD + 5 type-2 sub-IFDs; also OM Digital / OM System |
| Panasonic | `Vendor::Panasonic` | |
| Pentax | `Vendor::Pentax` | Also matched on `RICOH` |
| Samsung | `Vendor::Samsung` | |
| Sigma | `Vendor::Sigma` | Also matched on `FOVEON` |
| Sony | `Vendor::Sony` | |

Vendors intentionally absent (e.g. GoPro, which uses the 4CC `GPMF` format
rather than a standard tag-based maker note) return `None` from
`Vendor::from_make`.

## How to enable it

This crate is not meant to be depended on directly. Enable it through the
`exiftool-tables` feature on the consuming crate:

```text
# In revelo-parsers-tag, revelo-cli, or the revelo facade:
cargo build --features exiftool-tables
```

```text
# Install the CLI with ExifTool-grade maker-note depth:
cargo install revelo-cli --features exiftool-tables
```

The feature is off by default. The default build uses hand-written, clean-room
tables and stays fully BSD-2-Clause.

## Regenerating tables

The tables are generated by `codegen/extract.pl` against an installed ExifTool:

```text
perl crates/revelo-exiftool-tables/codegen/extract.pl
```

Only the extractor script is original revelo work (BSD-2-Clause). Its output —
everything under `src/generated/` — is a derivative of ExifTool and carries the
GPL/Artistic terms.
